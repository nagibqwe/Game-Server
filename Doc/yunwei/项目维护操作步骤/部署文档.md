### GM后台配置

- 使用TzjBackend.zip包解压到tomcat应用目录
- 修改配置文件 九 零一 起玩 www.90 175.com

** 替换以下配置文件中的{{}}**
- MYSQL_HOST : 数据库连接地址
- PORT： 数据库端口
- BACKEND_DATABASE: GM后台数据库名
- USERNAME: 用户名
- PASSWORD：密码
- LOGIN_DATABASE： 登录服逻辑库
- LOGIN_LOG_DATABASE： 登录服日志库

```
cd /usr/local/tomcat/webapps/TzjBackend/WEB-INF/classes/custom

cat db_backend.properties
db.backend.url=jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{BACKEND_DATABASE}}?autoReconnect=true&useUnicode=true&characterEncoding=UTF-8
db.backend.username={{USERNAME}}
db.backend.password={{PASSWORD}}
db.backend.validationQuery=select 1
db.backend.maxActive=100

cat db_loginlog.properties 
db.loginLog.url=jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{LOGIN_LOG_DATABASE}}?autoReconnect=true&useUnicode=true&characterEncoding=UTF-8
db.loginLog.username={{USERNAME}}
db.loginLog.password={{PASSWORD}}
db.loginLog.validationQuery=select 1
db.loginLog.maxActive=100

cat db_login.properties
db.login.url=jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{LOGIN_DATABASE}}?autoReconnect=true&useUnicode=true&characterEncoding=UTF-8
db.login.username={{USERNAME}}
db.login.password={{PASSWORD}}
db.login.validationQuery=select 1
db.login.maxActive=100
```

- 启动tomcat并检查日志，请确保服务可用

### 游戏服以及跨服配置

- 使用gameserver程序包部署
- 导入需要的sql文件，tzj_game，tzj_game_log
- 修改配置文件 config/server-config

- MYSQL_HOST:数据库地址
- PORT：数据端口
- GAME_DATABASE:逻辑库名
- USERNAME:数据库用户
- PASSWORD:数据库密码
- LOGIN_DATABASE:登录服逻辑库名
- LOG_DATABASE:日志库名
- GSID_LIST:合服列表,多个GSID使用,分割
- OPENTIME:开服时间 
- GM_HOST：GM地址
- GM_PORT: GM端口
- GM_DIR: GM访问URL
- AUTH_URL: 登陆验证url
- KEY:验证key
- TYPE：服务器类型
- PUBLIC_HOST：公共服IP
- PUBLIC_PORT：公共服端口
- SERVERNAME:服务器名
- GAME_PORT：对外提供的端口
- GSID:全服唯一ID，分配根据平台服规划
- BACKEND_PORT:与GM通信端口
- RECHARGE_PORT:充值回调端口

- biBakDir: BI日志备份目录
- openNet：开启日志标签 0关闭所有日志/1仅开启文本日志/2仅开启网络日志/3开启所有日志 建议配置值为2
- sysLogHost: bi日志接收地址
- sysLogPort: bi日志接收端口
- sysLogProtocol: bi日志传输协议

- bi4399Dir: 4399BI日志备份目录
- bi4399OpenNet: 4399BI开启日志标签 0关闭所有日志/1仅开启文本日志/2仅开启网络日志/3开启所有日志 建议配置值为2
- bi4399Host: 4399接收BI日志地址
- bi4399Port：4399BI日志接收端口
- bi4399Protocol: 4399BI日志传输协议
- bi4399Sql: 4399BI数据日志是否开启

## 两个BI日志都需要开启，BI日志备份目录建议不重复，建议日志开启标签值选2

```
<?xml version="1.0" encoding="UTF-8"?>
<!-- serverType字段服务器类型：0表示测试服，1表示正式游戏服，4表示跨服 -->
<server>
    <db-game url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{GAME_DATABASE}}?autoReconnect=true&amp;useUnicode=true&amp;characterEncoding=utf8" username="{{USERNAME}}" password="{{PASSWORD}}"/>
    <db-public-data url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{LOGIN_DATABASE}}?autoReconnect=true&amp;useUnicode=true&amp;characterEncoding=utf8" username="{{USERNAME}}" password="{{PASSWORD}}"/>
    <db-log-info url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{LOG_DATABASE}}?autoReconnect=true&amp;useUnicode=true&amp;characterEncoding=utf8" username="{{USERNAME}}" password="{{PASSWORD}}"/>
    <serverIdList serverIdList="{{GSID_LIST}}"/>
    <openTime openTime="{{OPENTIME}}"/>
    <heartHttp host="http://{{GM_HOST}}:{{GM_PORT}}/{{GM_DIR}}/serverstate/checkstate?"/>
    <errorlog http="http://{{GM_HOST}}:{{GM_PORT}}/{{GM_DIR}}"></errorlog>
    <login-data verifyUrl="{{AUTH_URL}}"/>
    <privatekey privatekey="{{KEY}}"/>
    <serverType serverType="{{TYPE}}"/>
    <public publicIp="{{PUBLIC_HOST}}" publicPort="{{PUBLIC_PORT}}"/>
    <serverInfo serverName="{{SERVERNAME}}" port="{{GAME_PORT}}" serverId="{{GSID}}" platfrom="cn" lsId="28"/>TET
    <language language="cn"/>
	<backgrand port="{{BACKEND_PORT}}"/>
    <!--充值端口-->
    <recharge rechargePort="{{RECHARGE_PORT}}"/>
    <!--BI syslog config:openNet开启日志标签，0关闭所有日志/1仅开启文本日志/2仅开启网络日志/3开启所有日志-->
    <biSysLog biBakDir="E:\luan" openNet="0" sysLogHost="10.0.0.59" sysLogPort="6789" sysLogProtocol="TCP"/>
    <!--bi4399Sql 1开启4399的数据日志，0关闭-->
    <bi4399SysLog bi4399Dir="E:\4399" bi4399OpenNet="0" bi4399Host="msg.4399sy.com" bi4399Port="5935" bi4399Protocol="UDP" bi4399Sql="1"/>

</server>
```

- 使用 fightserver_gameserver_manager.sh 启动并检查日志

### 公共服配置

- 使用public包部署
- 导入需要的sql文件，tzj_public，tzj_public_log
- 修改配置文件 config/server-config
- MYSQL_HOST:数据库地址
- PORT:数据库端口
- PUBLIC_DATABASE：公共服逻辑库
- PUBLIC_LOG_DATABASE：公共服日志库
- USERNAME:用户名
- PASSWORD:密码
- GM_DIR：GM目录
- GM_HOST:GM地址
- GM_PORT:GM端口
- AUTH_URL：验证url
- KEY：验证key
- Type字段服务器类型：0表示测试服，1表示正式游戏服，2表示跨服
- BACKEND_PORT：GM通信端口
- PUBLIC_PORT：公共服监听端口
- GSID: 全球唯一GSID，分配根据平台服规划

```

<?xml version="1.0" encoding="UTF-8"?>
<!-- server Type字段服务器类型：0表示测试服，1表示正式游戏服，2表示跨服 3 是公共服-->
<server>
    <db-game url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{PUBLIC_DATABASE}}?autoReconnect=true" username="{{USERNAME}}" password="{{PASSWORD}}" />
    <db-log-info url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{PUBLIC_LOG_DATABASE}}?autoReconnect=true"  username="{{USERNAME}}" password="{{PASSWORD}}" />
    <heartHttp host="http://{{GM_HOST}}:{{GM_PORT}}/{{GM_DIR}}/serverstate/checkstate?"/>
    <errorlog http="http://{{GM_HOST}}:{{GM_PORT}}/{{GM_DIR}}"></errorlog>
    <login-data verifyUrl="{{AUTH_URL}}"/>
    <privatekey privatekey="{{KEY}}"/>
    <serverType serverType="{{TYPE}}" language="cn" />
    <backgrand port="{{BACKEND_PORT}}" />
    <serverInfo serverName="publicserver" port="{{PUBLIC_PORT}}" serverId="{{GSID}}" platfrom="cn" lsId="28" />
</server>

```

- 执行publicserver_manager.sh 启服并检查日志

- MYSQL_HOST:数据库地址
- PORT：数据端口
- LOGIN_DATABASE:登陆库
- LOGIN_LOG_DATABASE：登陆日志库
- USERNAME:用户名
- PASSWORD：密码
- KEY：验证key
- LOGIN_PORT：登陆端口

```
<?xml version="1.0" encoding="UTF-8"?>

<!-- verifyUrl字段正式验证地址：https://auth.funcell123.com/verify -->
<!-- isNeedVerify字段：0表示测试版版本不用验证，1表示正式版本都需验证-->
<!-- serverinfo 信息当前进程的监听端口 及服务器ID 信息-->

<server>
	<db-login-data url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{LOGIN_DATABASE}}?autoReconnect=true" username="{{USERNAME}}" password="{{PASSWORD}}" />
	<db-log-info url="jdbc:mysql://{{MYSQL_HOST}}:{{PORT}}/{{LOGIN_LOG_DATABASE}}?autoReconnect=true"  username="{{USERNAME}}" password="{{PASSWORD}}" />
	<privatekey privatekey="{{KEY}}"/>
	<serverInfo serverName="agentserver" port="{{LOGIN_PORT}}" serverId="{{GSID}}" platfrom="cn"  lsID="28"/>
</server>

```
